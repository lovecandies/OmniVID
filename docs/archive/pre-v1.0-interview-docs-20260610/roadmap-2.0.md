# OmniVid 2.0 路线

## 2.0 原则

1.0 之后不再把所有想法塞进当前版本。2.0 只做能明显提升生产化、演示质量或求职竞争力的模块。

## P0：生产化稳定性

- 登录、用户体系、知识库权限和多租户隔离。
- API Key 加密存储和密钥轮换。
- 上传分片、断点续传、大文件失败恢复。
- 后端 CI、Docker 镜像、部署脚本和环境变量模板。
- 结构化日志、请求 ID、链路追踪和错误告警。

## P1：AI 精度提升

- 接入真实外部 Embedding Provider：Qwen、OpenAI、BGE 服务。
- 外部 BGE reranker 生产化。
- ASR 模型升级到更高精度模型或云端 ASR。
- OCR 字幕强证据对齐从诊断能力升级为默认可配置流水线。
- 术语词库前端管理增强：批量导入、项目级词库、视频级词库。
- 人工字幕校正工作流：编辑字幕、保存版本、重新生成总结和向量索引。

## P1：知识库增强

- 知识库批量导入视频。
- 多知识库切换和权限管理。
- 跨视频观点对比报告。
- 引用聚合视图：按视频、主题、时间轴聚类。
- 知识图谱：人物、概念、技术栈、观点关系抽取。

## P2：异步架构升级

- 从本地 DAG 演进到 RocketMQ。
- 节点拆分：上传、音频抽取、ASR、总结、向量索引、OCR、Agent 记忆。
- 幂等消费、重复消费处理、死信队列和延迟重试。
- 任务编排可视化和失败节点手动补偿。

## P2：导出与内容资产

- 真实 PPTX 文件生成。
- Markdown/HTML 博客导出。
- 会议纪要 DOCX 导出。
- 思维导图导出。
- 摘要模板市场：面试复盘、课程笔记、播客总结、会议复盘。

## P2：URL 导入与插件

- 合规 Cookie 管理和浏览器登录态读取。
- URL 导入任务队列与失败重试。
- 浏览器插件：一键把当前页面视频加入 OmniVid。
- 平台差异诊断：B 站、YouTube、抖音、小红书分别给出可操作建议。

## P3：企业化

- 团队空间。
- 审计日志。
- 计费统计：ASR 时长、LLM token、向量数量、存储容量。
- 对象存储：S3/OSS/MinIO。
- GPU ASR worker。
- 横向扩展和压测报告。

## 2.0 不做的事

- 不做平台反爬绕过。
- 不做盗版下载或 DRM 绕过。
- 不把所有模型 provider 硬塞到 1.0 主链路。
- 不为了堆简历关键词引入和业务无关的中间件。

## 建议里程碑

| 里程碑 | 目标 |
| --- | --- |
| 2.0-alpha | 登录、多用户、API Key 加密、人工字幕编辑 |
| 2.0-beta | 外部 Embedding + BGE rerank + OCR 默认流水线 |
| 2.0-rc | RocketMQ 异步化、PPTX/DOCX 导出、部署脚本 |
| 2.0 | 可公开演示的生产化版本 |
