# 01. OmniVid 求职型架构蓝图

## 定位

OmniVid 第一阶段是一个求职型 Java 后端项目，核心目标是展示：

- 复杂业务建模能力。
- MySQL 索引、事务、锁和状态机能力。
- Redis 缓存、锁、限流和语义缓存能力。
- Java 并发与异步任务编排能力。
- Spring / MyBatis 工程化能力。
- AI Agent 与 RAG 落地能力。

产品功能服务于面试叙事，不为了“看起来大而全”增加无关复杂度。

## 核心业务流

```text
用户登录
  -> 上传本地视频
  -> 计算 MD5
  -> Redis SETNX 防重复提交
  -> MySQL 唯一索引兜底去重
  -> 创建 processing_job
  -> 本地 DAG 异步解析
  -> 抽音频 / ASR / 字幕切片 / 向量化 / 总结
  -> SSE 推送任务进度
  -> 用户查看总结和字幕
  -> 点击字幕跳转视频时间点
  -> Agent 跨视频问答并返回时间戳引用
```

## 模块拆分

### Auth 模块

业务职责：

- 轻量账号登录。
- 用户视频与知识库隔离。

面试钩子：

- Filter、Interceptor、AOP 区别。
- JWT / Session 选择。
- ThreadLocal 用户上下文。
- 登录接口限流。

### Video 模块

业务职责：

- 视频上传。
- MD5 指纹。
- 去重命中。
- 文件元数据维护。

面试钩子：

- 大文件流式处理，避免 OOM。
- MySQL 唯一索引。
- Redis 分布式锁。
- 幂等接口设计。

### Job 模块

业务职责：

- 维护解析任务状态。
- 按 DAG 节点推进。
- 失败重试。
- SSE 进度推送。

面试钩子：

- 线程池参数。
- 拒绝策略。
- 任务队列选择。
- 乐观锁和状态条件更新。
- RocketMQ 平滑升级。

### Transcript 模块

业务职责：

- 存储字幕时间片。
- 支持播放器时间轴跳转。
- 支持 Agent 检索。

面试钩子：

- 联合索引 `(video_id, start_ms)`。
- 覆盖索引。
- EXPLAIN。
- 深分页与范围查询。

### Summary 模块

业务职责：

- 生成核心观点、金句、博客大纲、会议纪要、思维导图大纲。
- 缓存高频总结结果。

面试钩子：

- Cache Aside。
- 缓存一致性。
- 热点 Key。
- 模板策略模式。

### Agent 模块

业务职责：

- 根据知识库多视频上下文回答问题。
- 调用字幕检索工具。
- 返回时间戳引用。
- 防止无来源幻觉回答。

面试钩子：

- RAG。
- Embedding。
- 向量相似度。
- Tool Calling。
- 短期记忆 Redis，长期记忆 MySQL。

## 阶段路线

### Phase 1: 后端求职闭环

1. 用户登录。
2. 本地视频上传。
3. MD5 去重。
4. MySQL 状态机。
5. Redis 锁和任务进度。
6. 本地 DAG 任务推进。

验证：

- 上传同一个视频两次，只创建一条视频资产。
- 页面进度实时变化。
- 任务失败可看到失败节点和错误原因。

### Phase 2: 数据库与缓存亮点

1. 字幕表索引优化。
2. 视频列表分页优化。
3. Redis 限流。
4. 总结缓存。
5. EXPLAIN 证据截图或日志。

验证：

- 字幕查询命中联合索引。
- 高频问答被限流。
- 总结二次访问走缓存。

### Phase 3: AI Agent 加分项

1. 字幕向量化。
2. 知识库检索。
3. Agent 工具调用。
4. 回答带引用。
5. 语义缓存。

验证：

- 问答返回视频名和时间戳。
- 相似问题命中缓存。
- 无相关来源时拒答或提示证据不足。

## 简历主线

推荐项目名：

> OmniVid - 基于 Java 与 AI Agent 的长视频知识解析系统

推荐一句话：

> 该项目围绕长视频上传、异步解析、字幕检索与知识库问答，设计了基于 MySQL 状态机、Redis 防重限流和 Agent 工具调用的后端系统，重点解决长耗时任务可靠执行、重复解析成本高、AI 回答不可追溯等问题。
