# OmniVid v2.2 产品账号体系

## 范围

v2.2 只收束产品账号边界，不扩展新的视频 AI 能力。目标是让 OmniVid 从“个人演示项目”升级为“可公网使用的多用户产品雏形”。

## 模块边界

- `auth/`：注册、登录、邮箱验证、忘记密码、修改密码、当前用户状态。
- `account/`：当前用户自己的 Session、配额、数据导出、账号注销。
- `quota/`：视频数量、知识库数量、存储空间限制与用量统计。
- `admin/`：管理员查看用户、任务、失败记录、资源占用，并处理异常任务。
- `security/`：Session、CSRF、限流、管理员权限和密钥明文隔离。

## 黑盒验收

1. 普通用户注册后只能看到自己的视频、知识库、Provider 和 Agent 记录。
2. 普通用户可以请求邮箱验证、重置密码、修改密码、查看并踢出自己的 Session。
3. 普通用户超出视频数量、知识库数量或存储额度时，上传/创建会被明确拒绝。
4. 普通用户可以导出自己的账户数据，可以注销自己的账号。
5. 管理员可以查看用户列表、失败任务和资源占用，可以标记异常任务，但不能看到任何 API Key 明文。
6. 非管理员访问 `/api/admin/**` 和 `/api/vector-index/**` 会返回 403。

## 面试钩子

- Spring Security：Session 登录、CSRF、权限表达式、管理员角色。
- Redis Session：登录设备列表、服务重启后 Session 仍可管理。
- MySQL：账号 Token、配额统计、资源归属校验、软删除。
- Redis 限流：登录失败、接口限流、验证码触发阈值。
- 数据安全：Provider Key 加密存储，管理员接口只返回 masked key。
- SaaS 多租户：所有业务表都通过 `user_id` 或资源归属间接隔离。
